将外部 Markdown 笔记批量导入当前 Astro 博客仓库的标准化流程。

## 前置准备：了解博客规范

在动手之前，先扫描仓库获取以下信息（这些是自动从代码中推导的，不要问用户）：

1. **文章目录**：找到存放 `.md` 文章的目录（通常是 `src/content/posts/`）
2. **Frontmatter schema**：读取 `src/content.config.ts`（或类似的内容配置文件），了解必填和可选字段
3. **现有命名规则**：列出已有文章文件名，归纳命名模式（如 `Category-TopicPascalCase.md`）
4. **标题格式**：检查已有文章的 `title` 字段格式（如 `【标签】描述`）
5. **已有标签列表**：提取所有已使用的 `tags` 值，归类整理

这些信息决定了后续所有步骤的格式要求。

## 执行流程

### 第一步：复制文件

将源 Markdown 文件直接复制到文章目录，不要做任何内容修改。

### 第二步：添加 Frontmatter

为每个文件添加 frontmatter 头部，规则：

- **title**：遵循已有标题格式（如 `【标签】描述`），从正文 H1 提取并按规则包装
- **published**：使用当天日期，格式 `YYYY-MM-DD`
- **description**：从正文内容提炼一句话摘要
- **lang**：根据文章语言设置（中文文章填 `zh`）
- **tags**：从已有标签列表中选择最匹配的，可多选

同时**删除正文中的 H1 标题行**（`# 标题`），因为 frontmatter 的 `title` 会负责渲染页面标题。保留 H1 下方的引用块（`>`）、正文等内容。

如果正文中包含手动目录（`## 目录` 及其下方的链接列表），将其**注释掉**（用 `<!-- -->` 包裹），因为博客主题会自动生成目录，手动目录是冗余的。

### 第三步：重命名文件

按已有命名规则重命名文件，常见模式：
- 英文 PascalCase，无空格无中文
- `CategoryPrefix-TopicName.md` 格式
- 与同类别已有文件保持前缀一致

### 第四步：确认与预览

向用户展示每个文件的：
- 文件名（新旧对比）
- frontmatter 内容
- 标签选择

然后启动开发服务器预览效果。

## 注意事项

- 标签必须从已有列表中选取，不要随意创造新标签。如果觉得需要新标签，先询问用户
- 文件名只用英文字母、数字和连字符
- frontmatter 日期不要带时间部分，与已有文章保持一致（`YYYY-MM-DD`）
- 如果用户提供了多个文件，并行处理但统一展示结果
- 如果用户对标签、标题、文件名有修改意见，立即调整不要重新走完整流程
