# Jekyll to Astro Migration Design

## Overview

Migrate the personal blog from Jekyll (TeXt Theme) to Astro (Retypeset Theme). The blog is hosted at https://bingqiangzhou.github.io and deployed via GitHub Pages.

**Current state:** Jekyll + TeXt Theme v2.2.4, 37 posts (2019-2020), 64 images (~10MB), 7 posts with LaTeX math.

**Target state:** Astro + Retypeset Theme, GitHub Pages deployment via GitHub Actions, all content preserved.

## Architecture

### Project Structure


### Tech Stack

- **Astro** - Static site generator
- **UnoCSS** - Atomic CSS framework
- **TypeScript** - Type safety
- **pnpm** - Package manager
- **KaTeX** - Math rendering (via remark/rehype plugins)
- **Retypeset Theme** - Base theme (radishzzz/astro-theme-retypeset)

## Content Migration

### Posts (37 total)

| Aspect | Jekyll (Source) | Astro/Retypeset (Target) |
|---|---|---|
| Location | `_posts/YYYY-MM-DD-Title.md` | `src/content/posts/Title.md` |
| Filename | Date prefix required | No date prefix; date in frontmatter |
| Frontmatter | `title`, `tags` | `title`, `published`, `description`, `tags`, `lang` |
| Excerpt | `<!--more-->` separator | `description` field (auto-extracted) |
| Math syntax | `$$...$$` block, `$...$` inline | Same syntax (KaTeX compatible) |
| Image refs | `![](/assets/images/YYYY/YYYYMMDD/xxx.png)` | `![](/assets/images/YYYY/YYYYMMDD/xxx.png)` (paths preserved) |

### Migration Steps for Each Post

1. Copy `_posts/YYYY-MM-DD-Title.md` → `src/content/posts/Title.md`
2. Extract date from filename → add `published: YYYY-MM-DD` to frontmatter
3. Extract text before `<!--more-->` → add as `description` field
4. Add `lang: zh` to all posts
5. Keep all Markdown body content unchanged
6. Remove `<!--more-->` separator (no longer needed)

### Images (64 files, ~10MB)

- Preserve entire `assets/images/` directory structure under `public/`
- All existing image paths (`/assets/images/YYYY/YYYYMMDD/xxx.png`) remain valid
- No path changes required in any post

### Other Pages

- `about.md` → `src/content/about/zh.md`
- Archive page is auto-generated by Retypeset
- ToDo page: evaluate whether to keep or drop

### Features

**Math (KaTeX):**
- 7 posts with LaTeX math content
- Retypeset has built-in KaTeX support via remark/rehype plugins
- `$$...$$` and `$...$` syntax is directly compatible, zero changes needed

**Code Highlighting:**
- Replaces Jekyll's rouge + tomorrow-night-eighties
- Retypeset uses built-in syntax highlighting via Astro

**Dark Mode:**
- Built-in light/dark theme toggle in Retypeset
- No extra configuration needed

**Tags:**
- Existing tags preserved: 日常小结系列, 论文阅读笔记系列, 读书笔记系列, etc.
- Retypeset auto-generates tag aggregation pages

**i18n:**
- Simplified to Chinese-only mode
- Avoid `/zh/` URL prefix for single-language site
- May need minor config adjustment to Retypeset's i18n routing

### Features NOT Migrating

These were configured but never activated in Jekyll:

- Comments (Disqus/Gitalk/Valine) - provider: false
- Sharing (AddThis/AddToAny) - provider: false
- Analytics (Google Analytics) - provider: false
- Pageview (LeanCloud) - provider: false
- jemoji plugin - Astro handles emoji natively

## Deployment

### GitHub Actions Workflow

```yaml
Trigger: push to master
Steps:
  1. Checkout code
  2. Setup pnpm
  3. pnpm install
  4. pnpm build (outputs to dist/)
  5. Deploy to GitHub Pages via actions/deploy-pages
```

- `astro.config.ts` sets `site: https://bingqiangzhou.github.io`
- Pure Node.js build, no Ruby/Gem dependency
- Significantly faster build than Jekyll

### Astro Config (astro.config.ts)

- `site: https://bingqiangzhou.github.io`
- `output: 'static'` for GitHub Pages compatibility
- Integrations: UnoCSS, KaTeX plugin, sitemap

## Migration Execution Plan

### Phase 1: Project Initialization

- Clone Retypeset theme to temporary directory
- Integrate theme files into current repo (preserve `_posts/` and `assets/images/`)
- Remove Jekyll-specific files (Gemfile, _layouts/, _includes/, _sass/, _data/, etc.)
- Install dependencies with pnpm

### Phase 2: Content Migration

- Migrate 37 posts from `_posts/` to `src/content/posts/`
- Transform frontmatter for each post (extract date, add description and lang)
- Copy `assets/images/` to `public/assets/images/` (preserving directory structure)
- Migrate about page to `src/content/about/zh.md`

### Phase 3: Site Configuration

- Configure `src/config.ts` (title, author, URL, navigation)
- Configure `astro.config.ts` (site URL, GitHub Pages settings)
- Simplify i18n to Chinese-only mode
- Verify KaTeX, code highlighting, dark mode work correctly

### Phase 4: Validation & Testing

- Run `pnpm dev` to start dev server
- Check: homepage, post list, individual posts (especially 7 math-heavy posts)
- Check: tag pages, archive page, about page
- Check: dark mode toggle
- Check: all 64 images load correctly
- Run `pnpm build` and verify zero errors

### Phase 5: Deploy

- Create `.github/workflows/deploy.yml`
- Push to master branch
- Verify live site at https://bingqiangzhou.github.io

## Risk Controls

- Full git history preserved; Jekyll state recoverable at any time
- Content migration is copy + transform; original `_posts/` only deleted after validation
- Each phase is a separate commit, independently verifiable
- `pnpm build` serves as a validation gate before deployment
